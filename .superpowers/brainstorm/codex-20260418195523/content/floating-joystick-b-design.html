Design: tap implies intent, drag moves

This keeps the game one-hand friendly while preserving the current movement/action model.

Touch behavior state flow
1. Touch begins

Place joystick at thumb

The base appears centered on the first touch position, unless the touch started on an existing UI button or panel.

2. Movement threshold

Drag becomes movement

If the thumb moves beyond a small threshold, the touch is treated as joystick input and updates player movement.

3. Short tap

Tap means action intent

If the touch ends without enough drag, it sends one action pulse, equivalent to tapping ACT / pressing E.

What changes
  • The fixed bottom-left joystick becomes a floating joystick.
  • The joystick can start wherever the player touches the canvas.
  • A short tap fires one action pulse, so the tap location represents intent.
  • The ACT button stays available for explicit gather/interact timing.
  • Panel buttons, combat buttons, and modals keep their existing tap behavior.
Guardrails
  • Ignore floating joystick starts on buttons, panels, inputs, and combat menus.
  • Use a small movement threshold so ordinary taps do not accidentally move.
  • Hide the joystick immediately after release.
  • Only one active joystick touch at a time.
  • Do not change desktop keyboard controls.

Processing Power Codebase Map

A visual explanation of how this browser 3D idle RPG is wired together.

Runtime Flow
HTML Shell

index.html

Defines the canvas, HUD panels, combat overlay, mobile controls, styles, import map, and module entry point.

Coordinator

js/main.js

Creates every system, wires keyboard/click handlers, switches zones, handles interactions, and runs the animation loop.

Frame Loop

gameLoop()

Updates player, systems, combat, gathering, portals, HUD, camera, and renderer once per frame.

Main Ownership Boundaries

Scene

SceneManager owns Three.js scene, renderer, orthographic camera, resize handling, camera follow, and render.

Environment builds zones, terrain, props, portals, stations, collision circles, trees, rocks, and spawn metadata.

Entities

Player owns mesh, position, movement, facing, steps, terrain effects, and node gathering state.

EntityManager owns resource nodes and enemies, including aggro-triggered combat starts.

Systems

Small state machines for PP, stats, combat, inventory, crafting, drones, pedometer, saves, achievements, minigame, drill, ascension, and offline rewards.

Input Path

Keyboard: main.js maintains a keysDown set.

Mobile: TouchInput exposes dx, dz, isMoving, and actionPressed.

Player: Player.update(keysDown, delta, touchInput) consumes both paths and moves the character.

UI Path

HUD renders most non-combat panels: stats, inventory, crafting, drones, equipment, pedometer, achievements, ascension, drill, minigame.

CombatUI renders the combat overlay, action buttons, skill/item menus, HP bars, FP ring, and combat log.

Persistence Boundary

SaveSystem snapshots the live systems into JSON: zone, player position, PP, stats, inventory, pedometer tracks, drones, equipment, game stats, achievements, minigame, ascension, and auto-combat.

Loading applies JSON back into the existing system instances, then returns the zone and player coordinates so main.js can switch zones and teleport the player.

Growth Pressure

The architecture is understandable, but main.js, Environment.js, and HUD.js are the three largest files and are doing the most coordination work. Those are the likely refactor targets as the game grows.
